Recipe App

TL;DR

What: A mobile recipe application built using Xamarin

Why: To store, create, and edit personal and family favorite recipes

How: By using a MVVM architecture, Xamarin Forms, and SQLite Db

Why Xamarin Forms?

When developing mobile applications, there were two different paths I was choosing from. I could create platform specific apps using Java/Kotlin for Android and Objective-C or Swift for iOS/Apple or I could use Xamarin Forms and not have to worry about the extra overhead of learning those platform specific languages. For this application I chose Xamarin. I am aware of the pros and cons for each route, but the reasoning behind my choice is as follows

  • Shared UI - Xamarin allows me to share UI design and layout across platforms. This extends to windows too I beleive!
  • Shared logic - Just as a shared UI, my business logic can be shared across platforms as well. This reduces development time and cost dramatically
  • Visual Studio and C# - Instead of tackling multiple languages like Java/Kotlin/Objective-C/Swift I can develop in Visual Studio and C# and Xamarin takes care of the rest

MVVM

The MVVM architecture stands for Model - View - ViewModel. I especially like using it with when developing mobile applications because its main focus is to separate the business logic from the presentation logic. This all makes sense to me, and I like to have some code organization in my project.

  • Model - These are your objects being acted on. In the case of this app, my models are Recipe and Ingredients. both of which have their own properties associated.
  • View - This is where the presentation logic lives. In Xamarin, this is a .xaml file. Some of my views are MainPage.xaml and DetailsPage.xaml
  • ViewModel - This is basically the messenger between the model and view. This is where databinding and command events are made. The rule of thumb here is that for every view I have, I have a viewmodel that goes with it. For example, my MainPage.xaml has a MainPageViewModel.cs that takes care of command events and databinding for that view.

The Recipe App

The app I created using these previously mentioned frameworks and architectures is a pretty simple one. I wanted to be able to view, create, edit, and delete my favorite recipes. This app accomplishes these use cases. Here are some screen shots of how it looks

Main Page

The main page is where all of my favorite recipes can be viewed. My View has databinding on all of my stored recipes so that if and when I add one, it can immediately be seen on the home screen. I displayed them using xaml ListView.

Add Recipe Page

This is how and where you create a new recipe. It has some basic attributes like title, instructions, and ingredients. When adding a new ingredient, it will automatically appear below the save button letting you know which ingredients you've added. What you are seeing here are a bunch of Label and Entry xaml properties on my view all with databinding.

Detail Page

This is where a user can access their recipes to edit or remove them. This is all made possible with databinding between my View and ViewModel.
