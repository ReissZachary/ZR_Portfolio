PHP + Laravel Api

TL;DR

What: A Super Smash Brothers api built using Laravel and PHP that executes CRUD operations on a Super Smash Brothers database

Why: To create a rest api using a popular PHP framework

How: By taking advantage of Laravels MVC structure, controllers, built in database, and built in routing features

PHP + Laravel

Php is one of the most popular languages today especially among web developers. Together with its simplicity, scalability, community, and flexibility php offers diverse frameworks like Laravel.

Laravel is one of the most popular PHP frameworks used to build web apps. I have really enjoyed learning and using Laravel simply because of its scalability, familiarity, and features. There are many other frameworks to choose from, but the reason I chose Laravel is because of the points above. Above all, Laravel is the most widely used.

Summary

This is a similar api that I created in Django. To check out that blog post go HERE. What this api does is allows a user to create Super Smash characters then be able to execute CRUD operations on those characters. It utilizes a built in database where the characters are stored.

Routes

Routes are a crucial part in creating a RESTful api in Laravel. Routes are basically the endpoints that will be hit by POST, GET, UPDATE, etc. requests. In Laravel, you can group routes, name them, apply filters, and bind bind model data to them. Below is my route example for this app.

Couple things to point out. The Route::group attribute is where I created a group with a common prefix of api/v1 This means that each endpoint will be prefixed with api/v1.

The Route::resource is my endpoint. The first parameter character is the endpoint and the CharacterController is telling Laravel which controller to go to when this endpoint is hit. This leads us to our controller.

Controller

Simply put, Laravels controllers are very similar to other RESTful controllers like .NetCore for example. This controller enables me to separate logic behind serving the GET or POST requests. Below is one method in my CharacterController.

This store method is similar to a create method in other languages. The Request parameter is the body of the request that will hold attributes of a Character. The if($character->save() check is making sure the character could be created and if it was successful, it returns a json response with a message, link to newly created character, and a 201 response code.
